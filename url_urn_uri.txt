		URL (URI, URL, URN) универсальный указатель

			URL
URL(Uniform Resurs Locator)- указывает на место расположения (адрес) в сети интернет 
уникального ресурса( HTML-страница, CSS-файл, изображение и т.д. )
https://developer.mozilla.org
https://developer.mozilla.org/ru/docs/Learn/
https://developer.mozilla.org/ru/search?q=URL


			URN
URN(Uniform Resurs Name)- указывает имя файла или ресурса в сети
URN является частью концепции URI 
URN-  это постоянная последовательность символов, идентифицирующая 
абстрактный или физический ресурс


			URI
URI(Uniform Resurs Identfier)- указывает и имя файла и его адрес
(англ. Uniform Resource Identifier) — единообразные идентификаторы ресурса
			URI=URL+URN



			Структура URL

http://www.example.com:80/path/to/myfile.html?key1=value1&key2=value2#SomewhereInTheDocument
протокол(HTTP или HTTPS)//(хост или IP адрес):(номер порта)/(путь)?(параметр запроса)#(якорь)

протокол и хост(доменное имя) обязательны, остальные параметры факультативны

http:// 
это протокол. Он отображает, какой протокол браузер должен использовать. 
Обычно это HTTP-протокол или его безопасная версия - HTTPS. 
есть еще 
mailto:
ftp: 

www.example.com 
это доменное имя.(хост) Оно означает, какой веб-сервер должен быть запрошен. 
В качестве альтернативы может быть использован и IP-адрес

:80 
это порт. Он отображает технический параметр, используемый для доступа к 
ресурсам на веб-сервере. Обычно подразумевается, что веб-сервер использует 
стандартные порты HTTP-протокола (80 для HTTP и 443 для HTTPS) для доступа к своим ресурсам

/path/to/myfile.html
это адрес ресурса на веб-сервере

?key1=value1&key2=value2 
это дополнительные параметры, которые браузер сообщает веб-серверу. 
Эти параметры - список пар ключ/значение, которые разделены символом &.

#SomewhereInTheDocument 
это якорь на другую часть того же самого ресурса. 
Якорь представляет собой вид "закладки" внутри ресурса, которая переадресовывает браузер 
на "заложенную" часть ресурса.
