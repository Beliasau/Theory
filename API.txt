		API (Application Programming Interface — «программный интерфейс приложения»)
описание способов взаимодействия одной компьютерной программы с другими.
API — это составляющая часть сервера, которая получает запросы и отправляет ответы,
и является интерфейсом через который программы общаются между собой



API (Application Programming Interface — программный интерфейс приложения, или интерфейс 
программирования приложений) — специальный протокол для взаимодействия компьютерных программ, 
который позволяет использовать функции одного приложения внутри другого.





API (Application programming interface) — это контракт, который предоставляет программа. 
«Ко мне можно обращаться так и так, я обязуюсь делать то и это».
Соответственно, API отвечает на вопрос “Как ко мне, к моей системе можно обратиться?”, и 
включает в себя:
	саму операцию, которую мы можем выполнить,
	данные, которые поступают на вход,
	данные, которые оказываются на выходе (контент данных или сообщение об ошибке).




когда компания предлагает своим пользователям API, это просто означает, что она 
создала ряд специальных URL, которые в качестве ответа возвращают только данные.
Такие запросы часто можно отправлять через браузер. Так как передача данных по протоколу HTTP 
происходит в текстовом виде, браузер всегда сможет отобразить ответ. Например, через браузер 
можно напрямую обратиться к API GitHub (https://api.github.com/users/petrgazarov), причём без 
маркера доступа, и получить ответ в формате JSON.
Браузер отлично отображает JSON-ответ, который вполне можно вставлять в код. 
Из такого текста достаточно просто извлечь данные, чтобы использовать их по своему усмотрению.
